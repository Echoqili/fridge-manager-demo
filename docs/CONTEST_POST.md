**标题：【生活娱乐赛道】鲜知 fridge —— 拍一拍冰箱，AI 帮你配一桌菜**

## 1. Demo 简介

**是什么**：鲜知 fridge 是一个 AI 冰箱管家 Web 应用，通过拍照识别、智能库存管理和 AI 菜谱推荐，帮助用户减少食材浪费、优化膳食搭配。

**面向谁**：25-40 岁城市家庭主厨、独居白领、注重健康饮食的年轻人。

**主要功能**：

1. **📸 拍照识食材** — 上传冰箱照片，AI（GPT-4o）自动识别食材种类、数量与存放位置，识别失败自动降级到本地推断
2. **🧊 智能库存管理** — 可视化冰箱分区（冷藏/冷冻/储物柜），临期食材自动标红提醒，支持分类筛选
3. **✨ AI 菜谱推荐** — 根据现有食材智能匹配菜谱，优先消耗临期食材，包含完整烹饪步骤和营养信息
4. **📊 营养洞察** — ECharts 可视化分类统计 + AI 个性化饮食建议

## 2. Demo 创作思路

**灵感来源**：

每次打开冰箱都在发愁"今天吃什么"，经常发现食材过期了才想起来，买菜时也不清楚家里还有什么。据统计，中国家庭每年浪费的食物约 1700 万吨，其中大量源于冰箱管理不当。如果冰箱能"开口说话"，告诉我该做什么菜、什么快过期了，就能大大减少浪费。

**想解决的问题**：

- 食材过期浪费 — 用户忘记冰箱里有什么，食材放到过期才扔掉
- 不知道做什么菜 — 打开冰箱发呆 10 分钟，最后还是点外卖
- 营养搭配不均衡 — 不清楚自己的饮食结构是否健康
- 重复购买 — 买菜时不知道家里已有食材，重复采购

**为什么做这个方向**：

冰箱管理是每个家庭的日常痛点，但市面上的方案要么是简单的清单 App（手动录入太麻烦），要么是昂贵的智能冰箱硬件。用 AI 图像识别 + 菜谱推荐的方式，可以在不改变用户习惯的前提下，用一张照片解决"录入难"的问题，用 AI 解决"决策难"的问题。

### 踩过的坑

1. **GitHub API 推送大文件集**：90+ 文件的 commit 通过 MCP 推送时遇到 API 限制，最终通过分批创建 blob + 并行处理解决
2. **前后端分类体系不一致**：后端用 meat/dairy 6 类，前端用 protein 5 类，导致数据不匹配。通过统一为后端 6 类分类解决
3. **登录契约 Bug**：后端 TokenResponse 不返回 user 字段，前端期望 data.user，导致真实登录后被踢回登录页。通过后端添加 user 字段解决
4. **E2E 测试时序问题**：演示模式用 window.location.href 触发整页刷新，Playwright waitForURL 时序冲突。改用 React Router navigate 解决
5. **AI 菜谱排序冲突**：AI 生成菜谱与本地菜谱匹配数相同时排序不稳定。通过排序键 `(match_count, source == 'ai')` 确保 AI 优先
